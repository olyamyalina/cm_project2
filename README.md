# Минимальный CLI-прототип с конфигурацией

## Цель
Создать минимальное CLI-приложение на Python с настраиваемыми параметрами через CSV-файл.  
Приложение должно корректно обрабатывать ошибки и выводить параметры в формате ключ-значение.
Реализовать основную логику получения данных о зависимостях для их дальнейшего анализа и визуализации. 

---

## Описание

На этом этапе реализовано CLI-приложение, которое:

1. Считывает параметры из конфигурационного файла формата CSV (config.csv).
2. Поддерживает следующие настраиваемые параметры:
   - **package_name** - имя анализируемого пакета.
   - **repo_path** - URL репозитория или путь к локальному тестовому репозиторию.
   - **mode** - режим работы с тестовым репозиторием (local, remote или test).
   - **max_depth** - максимальная глубина анализа зависимостей.
3. При запуске выводит все параметры в формате **ключ = значение**.
4. Обрабатывает ошибки конфигурации:
   - отсутствие или пустое значение обязательных полей;
   - некорректный путь или URL репозитория;
   - неверный режим работы;
   - некорректное значение max_depth.
5. Извлекает прямые зависимости пакета Rust:
   - поддерживается локальный и удалённый режим работы (GitHub).
   - в удалённом режиме программа получает файл Cargo.toml из корня ветки main или master.
   - парсится секция [dependencies] и связанные с ней секции, чтобы собрать имена прямых зависимостей.
6. Выводит на экран список всех прямых зависимостей выбранного пакета.
7. Построение графа зависимостей:
   - cтроится граф зависимостей с учётом транзитивности (все прямые и косвенные зависимости);
   - используется алгоритм BFS (обход в ширину) без рекурсии, с учётом заданной максимальной глубины max_depth;
   - корректно обрабатываются циклические зависимости, которые помечаются как (cycle) в выводе;
   - поддерживается тестовый режим (mode = test), где вместо реального репозитория можно указать файл с описанием графа (test_graph.txt);
     • пакеты обозначаются заглавными латинскими буквами (A–Z)
     • формат строки: X: Y Z — пакет X зависит от Y и Z.
8. Граф выводится на экран в виде дерева с наглядной вложенностью, отражающей глубину зависимостей.
